# Tarifa Wind iCal

Generate and publish an automatically updated `.ics` calendar feed with the best wind windows for Tarifa, using Open-Meteo.

The feed creates calendar events only when your conditions are met, instead of filling your calendar with every forecast hour.

## Features

- Uses Open-Meteo Forecast API for wind speed, gusts, and direction.
- Uses Open-Meteo Marine API for wave height, period, and direction.
- Publishes automatically to GitHub Pages.
- Updates every hour via GitHub Actions.
- No API key required for typical non-commercial use.
- No third-party Python packages required.
- Includes profiles for `kite`, `wingfoil`, and `windsurf`.
- Adds a session score from `0` to `100`.
- Labels Tarifa wind as Levante or Poniente when applicable.

## Quick setup on GitHub

1. Create a new public GitHub repository, for example `tarifa-wind-ical`.
2. Upload all files from this project into the repository.
3. Go to **Settings ‚Üí Pages**.
4. Under **Build and deployment**, set **Source** to **GitHub Actions**.
5. Go to the **Actions** tab.
6. Open **Update and publish Tarifa wind iCal**.
7. Click **Run workflow**.
8. When it finishes, open the GitHub Pages deployment URL.

Your calendar feed URL will be:

```text
https://YOUR-USERNAME.github.io/tarifa-wind-ical/tarifa-wind.ics
```

Replace `YOUR-USERNAME` with your GitHub username.

## Add it to Google Calendar

1. Open Google Calendar.
2. In the left sidebar, click **Other calendars ‚Üí + ‚Üí From URL**.
3. Paste your `.ics` URL:

```text
https://YOUR-USERNAME.github.io/tarifa-wind-ical/tarifa-wind.ics
```

4. Click **Add calendar**.

Google Calendar may cache subscribed calendars, so updates may not appear immediately after every hourly run.

## Add it to Apple Calendar

1. Open Apple Calendar.
2. Go to **File ‚Üí New Calendar Subscription**.
3. Paste your `.ics` URL.
4. Choose an auto-refresh interval.

## Run locally

```bash
python3 tarifa_wind_ical.py
```

This creates:

```text
tarifa-wind.ics
```

You can open that file directly in Apple Calendar, Outlook, or import it into Google Calendar.

## Choose a profile

The default profile is `kite`.

```bash
PROFILE=kite python3 tarifa_wind_ical.py
PROFILE=wingfoil python3 tarifa_wind_ical.py
PROFILE=windsurf python3 tarifa_wind_ical.py
```

To change the GitHub Actions profile, edit `.github/workflows/update-ical.yml`:

```yaml
env:
  PROFILE: kite
```

## Default profile settings

| Profile | Minimum wind | Good | Excellent | Max gust | Max wave |
|---|---:|---:|---:|---:|---:|
| `kite` | 16 kt | 20 kt | 25 kt | 40 kt | 1.8 m |
| `wingfoil` | 12 kt | 16 kt | 22 kt | 38 kt | 2.2 m |
| `windsurf` | 18 kt | 24 kt | 30 kt | 45 kt | 2.0 m |

## Customize the criteria

All settings can be overridden with environment variables.

Example:

```bash
MIN_WIND_KT=18 GOOD_WIND_KT=22 EXCELLENT_WIND_KT=28 MAX_WAVE_M=1.5 python3 tarifa_wind_ical.py
```

Useful variables:

| Variable | Default | Description |
|---|---:|---|
| `SPOT_NAME` | `Tarifa` | Calendar location/name |
| `LATITUDE` | `36.0143` | Spot latitude |
| `LONGITUDE` | `-5.6044` | Spot longitude |
| `TIMEZONE` | `Europe/Madrid` | Forecast timezone |
| `FORECAST_DAYS` | `7` | Number of forecast days |
| `PROFILE` | `kite` | `kite`, `wingfoil`, `windsurf`, or `custom` |
| `MIN_WIND_KT` | profile-based | Minimum average wind to create an event |
| `GOOD_WIND_KT` | profile-based | Threshold for a Good label |
| `EXCELLENT_WIND_KT` | profile-based | Threshold for an Excellent label |
| `MAX_GUST_KT` | profile-based | Reject sessions above this gust level |
| `MAX_WAVE_M` | profile-based | Reject sessions above this wave height |
| `MIN_BLOCK_HOURS` | profile-based | Minimum consecutive usable hours |
| `WIND_SECTORS` | `60-130,240-300` | Accepted wind direction sectors in degrees |
| `MIN_SCORE` | `35` | Skip sessions below this score |
| `OUTPUT_FILE` | `tarifa-wind.ics` | Output calendar filename |

## Wind sectors

The default sectors are:

```text
60-130,240-300
```

This roughly means:

- Levante: east / east-northeast / east-southeast
- Poniente: west / west-southwest / west-northwest

To accept all wind directions:

```bash
WIND_SECTORS=0-360 python3 tarifa_wind_ical.py
```

## Published filename

The GitHub workflow publishes:

```text
public/tarifa-wind.ics
```

If you change the filename, update both:

```yaml
OUTPUT_FILE: public/tarifa-wind.ics
```

and the link in the generated `public/index.html` inside `.github/workflows/update-ical.yml`.

## Notes

Open-Meteo forecast data can change from run to run. Calendar apps may also cache subscribed `.ics` feeds, so Google Calendar may not refresh immediately after each hourly update.
